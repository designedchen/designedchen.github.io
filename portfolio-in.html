Inventory App

PROJECT OVERVIEW

Inventory app is a consumer based app that helps people to search for their inventory easily without getting fustrated. This project is a 6 week project where I will be doing the full ux design process.

Project Process

Project Scope

PROBLEM:
People always tend to forget things all the time.

USER NEEDS:
User needs to be able to find items easily without having to spend a too much time looking for their items.

Roles Played:
UX Designer and Strategist

Primary Stakeholder:

  • Product Manager
  • Tech Manager
  • Engineer
  • Business Owners

Tools Used:

  • Sketch
  • UI Toolkit
  • Invision
  • Craft

Technology Available

  • Material Design UI Toolkit
  • API (Developer)

Research


I will be doing research based on the resources and doing online survey and interview gather data for this project. The focus on the research is to gather as much data to identify and recognize any missing information that need to be more concrete.

Persona


During this stage of the process, I create 3 user persona for this project. It helps company to better understand their pain points and goals. This helps to define the stay focus and fully understand the business and user's need and goals.

User Flow


As a user, user would like an app to help them remember where his/her place items within in their personal inventory (essential and non essential items):
- Ability to search inventory based on category
- Ability to capture images and making a list of mapped items
- Ability to view a map of my items within it’s space/room
- Ability to share images
- Ability to assist you how to organize

Wireframing


On this phase, I started to low-fidelity wireframe based on the user flow we designed and working on the skeleton of the interface to ensure every pages meet the user goal and understanding the information architecture.

Wireframe Usability Testing


During this stage, we create a testing panel with 5-8 users on-site to see if they are capable to perform tasks based on the questions we provide to them. The goal of this testing is to observe user performing the task and understanding their pain point and happiness using this app.

Testing Questions

1.How do you search for inventory?
2.How do you capture photos of inventory?
3.What do you think about the snapshot confirmation page? Is it necessary?
4.Will you to figure out how to edit the inventory?
5.What do you think about the photo preview page?
6.Were you able to share your inventory to people?

Result

After this usability testing, we discover pain point on the main menu page that user has a doubt about how to search for inventory even though the button says inventory does not automatically will make user know that it is also function as search button. This bring up the the decision that we will remove the main menu page and have the dashboard as the main page. This will help user understands better.
In addition, testers also mention a doubt about the delete pop up message that was not clear if the delete button on photo preview page will delete the text or the image or both. It brought to our attention that we will need to write it clearly without any doubt.

High Fidelity Wireframing: Color Style


Based on the research, inventory app need to feel warm, welcoming atmosphere, minimalist, peace, stress free, plain, free, calm, mindful. As a result, we decided to look for inspiration from minimalistic interior design as well as Japanese interior and garden design. I discover most colors are either shades of black or white or beige. As a project team, we came to decision to choose beige as the color for creating high fidelity wireframe.

High Fidelity Wireframing: Design


Based on the low fidelity wireframing and testing result, we made few changes to the wireframe and decided to create a dashboard to make it feel like "Home".

High Fidelity Wireframe Testing


During this high-fidelity prototype stage, we assign a panel of user to see if they are capable to perform specific tasks based on the questions.

Testing Questions

1.On dashboard page, what are the tasks do you think you are able to achieve?
2.What do you think about the arrangement spacing and type size on the dashboard?
3.Will you to figure out how to edit or delete the inventory?
4.What is your pain point of this app?

Result

After the usability testing, user were able to identify and perform the set of tasks. There were few UI problems placement position that confuses user and I start wondering why things were place over there at the first place. The spacing and text was a little bit too tight and feeling like it is very busy which can may cause confusion. After this testing, I conduct few more testing like A-B testing on specifics UI and UI placement to build a high-fidelity prototype. In terms of pain point, user was able to use it after showing the instruction. Users did had a struggle at first because it is a learning curve for them.

High Fidelity Prototyping

After multiple testing stage at high fidelity wireframing. Our project meeting had to agree based on the results of the testing as well as making sure it meet both user and business's needs before handing this prototype to engineering team to make app prototyping.

Results


“I can’t believe how this inventory app had save my day trying to find my vintage glassware that I almost gave up.”

Challenge:

During the user flow and wireframing stage, we went through different concept pathway to get to what user are looking for. There were so many steps to get user’s goal which can be intimating and causes frustration. We had to figure out ways to get to user’s goals in 3-4 steps or less in the wireframing.

What I learned:

I have learned research can be very diversify and may become to out of scope of in a project. It may or may not lead to the consistent information to both user and business goals. Which is why having multiple meeting is also play an important roles to ensure and keeping on the right track in order to meet all goals. I also learned the frequently testing will help and improve solving problems that user encounter before passing the final prototype to engineer team.
